# 《笑傲江湖》知识图谱阶段二：在线通用构建设计

## 1. 目标与范围

阶段二把阶段一的只读教学演示扩展为可在线构建小说知识图谱的单机应用。用户上传 TXT 小说后，可以选择 OpenAI 兼容 API 或 Ollama，观察章节切分、片段抽取、实体消歧、本体校验、Neo4j 入图和质量报告的全过程。

阶段二必须交付以下闭环：

- 上传小说并创建隔离项目；
- 使用独立 Worker 执行可暂停、恢复、取消和重试的持久化任务；
- 通过统一接口支持 OpenAI 兼容 API、Ollama 和测试固定响应；
- 将合法实体、事实和证据幂等写入 Neo4j；
- 在构建页面显示实时进度、错误摘要和质量报告；
- 在图谱、故事线和问答页面切换并使用用户项目；
- 删除项目时清理 SQLite 元数据、上传文件和 Neo4j 图数据。

本阶段不包含账户体系、多人协作审核、Redis、PostgreSQL、多机 Worker、对象存储和复杂人工消歧工作台。阶段三再根据真实质量数据改进章节识别、消歧和审核体验。

## 2. 已确认约束

- 部署目标是单机 Docker Compose。
- 模型密钥只通过服务端环境变量配置，不进入前端、SQLite、Neo4j、任务事件或日志。
- 单个上传文件最大 20 MB，仅接受 TXT；自动识别 UTF-8、UTF-8-BOM 和 GB18030。
- 默认自动化验收使用固定模型响应；OpenAI 兼容 API 与 Ollama 使用契约测试；真实模型调用是可选冒烟测试。
- 采用垂直切片交付，每个切片同时包含 API、界面和自动测试。

## 3. 总体架构

### 3.1 Web 前端

React 前端新增构建工作台和项目切换器。前端只调用 FastAPI，不直接访问文件系统、Neo4j 或模型服务。任务进度优先通过 SSE 接收，连接中断后通过任务详情接口恢复状态。

### 3.2 API 服务

FastAPI 负责上传校验、项目管理、任务控制、模型配置摘要和 SSE 状态推送。API 只写入任务意图，不执行长时间模型调用。所有输入使用 Pydantic 和显式限制校验。

### 3.3 独立 Worker

Worker 是独立进程，通过 SQLite 原子领取任务和片段。它执行章节切分、模型抽取、归一化、实体消歧、本体校验和批量入图。API 与 Worker 不共享进程内状态，因此 Worker 重启后可根据持久化状态继续执行。

阶段二不引入队列中间件。SQLite 采用短事务、领取租约和单写者约束满足单机部署；接口边界保留未来迁移 PostgreSQL 与正式队列的空间。

### 3.4 存储职责

- SQLite：项目、文件元数据、章节、片段、任务、片段尝试、错误摘要、进度和质量统计。
- 受控文件目录：按项目保存规范化后的 UTF-8 文本，文件名由服务端生成。
- Neo4j：实体、事实、证据、抽取批次、置信度和审核状态。

所有数据均以 `project_id` 隔离。内置 `xiaoao` 项目保持只读，用户项目可以删除。

### 3.5 模型适配层

统一接口接收本体约束、片段文本和偏移信息，返回结构化候选实体、事实与证据。实现包括：

- OpenAI 兼容适配器；
- Ollama 适配器；
- 固定响应适配器，仅用于测试和本地确定性演示。

供应商差异封装在适配层。前端只选择公开的模型配置档案，不提交认证头、密钥或任意模型请求体。

## 4. 任务状态机

正常阶段为：

`QUEUED → SPLITTING → EXTRACTING → RESOLVING → VALIDATING → IMPORTING → COMPLETED`

控制及终止状态为 `PAUSED`、`CANCELLED` 和 `FAILED`。

状态转换规则：

- 暂停只阻止领取新片段；正在执行的单次模型调用完成后必须保存结果。
- 恢复从最后持久化阶段继续，不重复已成功片段。
- 取消停止后续处理，但保留上传文件和已经完成的数据。
- 重试只重置失败片段和受其影响的后续阶段。
- Worker 领取任务时写入租约；租约超时后其他 Worker 可以安全接管。
- 所有导入使用稳定标识和幂等写入，重复执行不得复制实体、事实或证据。

## 5. 在线构建数据流

1. API 流式读取上传内容，限制 20 MB，拒绝空文件、非 TXT 和不支持编码。
2. 服务端生成项目 ID、文件 ID、受控路径和 SHA-256 指纹，将内容规范化为 UTF-8。
3. Worker 识别章节；识别不到明确章节时回退为正文分段，不直接终止任务。
4. 长章节按语义段落切片，保留有限重叠、章节信息和原文绝对偏移。
5. 模型根据本体和 JSON Schema 输出候选实体、事实及局部证据偏移。
6. 系统把局部偏移换算为原文绝对偏移，并校验引文与源文本完全一致。
7. 归一化实体类型、关系方向和枚举值；未知类型、本体冲突和越界证据进入拒绝记录。
8. 先使用规范名称与确定性别名合并；无法可靠判定的同名实体保持为独立候选。
9. 合法事实跨片段合并，多条证据关联到同一稳定事实。
10. 批量幂等写入 Neo4j，并生成项目级质量报告。

## 6. API 契约

### 6.1 项目与上传

- `POST /api/projects/upload`：multipart 上传 TXT，并提交标题、模型配置档案 ID；返回项目和任务摘要。
- `GET /api/projects`：列出内置项目和用户项目。
- `GET /api/projects/{project_id}`：返回项目、构建状态和图谱统计。
- `DELETE /api/projects/{project_id}`：幂等清理项目；内置项目返回禁止删除错误。

### 6.2 任务

- `GET /api/jobs/{job_id}`：返回当前阶段、片段计数、调用统计和安全错误摘要。
- `GET /api/jobs/{job_id}/events`：SSE 推送单调递增的状态快照和事件 ID。
- `POST /api/jobs/{job_id}/pause`：请求暂停。
- `POST /api/jobs/{job_id}/resume`：恢复暂停或可恢复失败任务。
- `POST /api/jobs/{job_id}/cancel`：取消任务。
- `POST /api/jobs/{job_id}/retry`：重试失败片段。
- `GET /api/jobs/{job_id}/quality`：返回质量报告和拒绝原因聚合。

非法状态转换返回 `409` 和稳定错误码。不存在或不属于指定项目的资源返回 `404`，不得泄漏其他项目信息。

### 6.3 模型配置档案

- `GET /api/model-profiles`：返回供应商、基础地址、模型名、上下文限制和可用状态。

档案由环境变量或服务端配置生成。API 不返回密钥，也不提供写入密钥的接口。

## 7. 构建界面

构建页面由三个连续区域组成：

1. 新建项目：拖放 TXT、填写项目标题、选择模型供应商和模型配置档案；上传前显示格式与大小限制。
2. 构建监控：显示阶段时间轴、片段进度、成功与失败计数、调用次数、暂停、继续、取消和重试操作。
3. 质量结果：显示实体数、事实数、证据数、拒绝原因、失败片段和待消歧候选，并提供进入项目图谱的入口。

顶部项目切换器控制图谱、故事线和问答页面使用的 `project_id`。当前项目保存在 URL 或可恢复的应用状态中，不能依赖不可重建的浏览器内存。页面刷新后重新读取 API 状态。

## 8. 错误处理

错误分为：

- 输入错误：文件超限、空文件、编码失败，上传请求直接拒绝。
- 配置错误：模型地址不可达、密钥缺失、模型不存在，任务进入失败状态且不开始抽取。
- 可重试错误：限流、超时和临时网络故障采用指数退避，单片最多尝试三次。
- 永久错误：Schema 不合法、本体冲突和证据偏移越界记录拒绝原因，并继续处理其他片段。

用户界面显示可行动的错误摘要和稳定错误码。原始异常、认证头、密钥、完整模型响应和长篇原文不得进入 SSE 或客户端错误信息。

## 9. 安全与资源控制

- 上传文件名和目录由服务端生成，所有路径必须验证仍位于配置的数据根目录内。
- 文件流式计数，达到限制即停止读取，避免先完整载入内存。
- 模型输出视为不可信输入，仅接受 Schema 允许字段并设置实体、事实、证据和字符串长度上限。
- API 不接受任意文件路径、Cypher、认证头、密钥或供应商原始请求体。
- 日志使用结构化错误码，并对认证信息和文本内容脱敏。
- 单任务限制并发片段数、单次超时、最大重试和累计调用数。
- 删除项目使用项目级互斥和幂等清理步骤，禁止路径越界或跨项目图删除。

## 10. 测试与验收

### 10.1 单元测试

- 三种编码识别、章节切分、回退分段、重叠切片和绝对偏移换算；
- 状态机合法与非法转换、租约过期接管和重试计数；
- JSON Schema、本体规则、关系方向、证据引文和稳定标识；
- 模型适配器错误分类及日志脱敏；
- 20 MB 限制、空文件、错误扩展名和路径穿越。

### 10.2 集成测试

- 固定响应模型完成 TXT 到 Neo4j 的完整流水线；
- OpenAI 兼容 API 和 Ollama 使用本地假服务验证请求与响应契约；
- Worker 重启后继续未完成任务；
- 暂停、恢复、取消和失败片段重试；
- 项目隔离、幂等导入和删除清理。

### 10.3 浏览器端到端测试

- 上传小说、查看实时阶段、完成后进入该项目图谱；
- 质量报告显示实体、事实、证据和拒绝项；
- 项目切换后图谱、故事线和问答均使用正确项目；
- 页面刷新和 SSE 重连后任务状态保持一致。

### 10.4 完成门槛

- Docker Compose 冷启动后 `make verify` 全部通过；
- 自动化全链路不依赖付费模型或本地 Ollama；
- 真实 OpenAI 兼容服务和 Ollama 冒烟测试可通过环境开关单独执行；
- 密钥扫描确认密钥不出现在前端、SQLite、Neo4j、日志和任务事件中；
- 上传、任务和图查询的项目隔离测试通过。
